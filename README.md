# Metric-service app
## Описание приложения
Это простой сервис, позволяющий получать, накапливать, отображать
данные от iot устройств путем http запросов, используя фреймворк
flask

Поскольку уровень доступа к сервису состоит из 3 классов
(viewer, user, admin). Далее будут перечислены возможности доступа
к сервису, начиная с самого низкого класса, подразумевая, что 
следующий класс имеет возможности доступа предыдущего.

### Viewer
Данный класс предназначен только для просмотра накопленных данных,
не имеет возможности менять какие-либо параметры, разве что кроме 
личной информации (пароль и т.п.). 
Возможности:
* Просмотр накопленных данных
* Все функции отображения и агрегирования данных, доступные сервисом
* Создание отчетов (pdf) о данных

### User
В данном классе пользователь может настраивать все параметры, 
связанные с получением и хранением данных, то есть:
* Добавлять/редактировать/удалять метрики системы
* Управлять жизненным циклом метрик (время хранение и т.п.)
* Добавлять пользователей класса Viewer, для просмотра своих данных

### Admin
Данный класс подразумевает полное управление сервисом, к тому же,
единственный класс, создающий пользователей и присвоения им классов. 
Система ручного управления пользователями используется для простоты
добавления пользователей в систему и тщательным контролем доступа 
к сервису.


## Техническое описание
Весь сервис опирается на взаимодействии с базой данных, в данном 
случае с sqlite (при необходимости можно перейти на postgres).
Реализованы следующие таблицы, необходимые для базового функционала.

### Таблица Users
* id (series): порядковый номер пользователя
* role (tinyint): 0 - admin, 1 - user, 2 - viewer
* login (text): логин пользователя
* hash (text): хеш пароля пользователя
* description (text): описание пользователя

### Таблица Metrics
* id (series): порядковый номер метрики
* name (text): название метрики
* type (char(3)): тип метрики (int, flt, dbl, str, bln)
* duration (int): длительность хранения данных в днях

### Таблица Metric_Owners
* metric_id (int)
* user_id (int)

### Таблица Data_int
* time (timestamp): время записи
* metric_id (int): id метрики
* value (int): int данные метрики в момент записи

### Таблица Data_flt
* time (timestamp): время записи
* metric_id (int): id метрики
* value (real): float данные метрики в момент записи

### API
Методы доступа:
* login: точка входа в сервис

Методы работы с информацией пользователя:
* user_add: добавление пользователя
* user_get: получение информации о пользователе
* user_update: обновление информации о пользователе (пароль, описание)
* user_delete: удаление пользователя

Методы работы с метриками:
* metric_add: добавление метрики
* metric_get: получение информации о метрике
* metric_update: обновление информации о метрике (название, владельцы, длительность хранения данных)
* metric_delete: удаление метрики и всех данных о ней

Методы работы с данными:
* data_add: добавление записи какой-то метрики
* data_get: получение записей (параметры записи в json)
* data_delete: удаление записей
